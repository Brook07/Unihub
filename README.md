## The Problem

Students currently juggle Google Classroom, email, Discord, WhatsApp, Class Representatives, and university portals just to stay informed — different courses use different tools, with no integration between them. This causes missed announcements, forgotten deadlines, lost resources, and constant app-switching.

## The Solution

**UniHub** centralizes everything a student needs — announcements, assignments, resources, timetable, calendar, messaging, study groups, and events — into one dashboard, so there's a single place to check instead of ten.

---

## Core Features

| Module | What it does |
|---|---|
| **Dashboard** | Today's classes, deadlines, announcements, and quick actions in one view |
| **Courses** | Dedicated workspace per course (assignments, notes, slides, discussion) |
| **Assignments** | Track, submit, and get feedback, with clear status (`Pending`/`Submitted`/`Graded`/`Overdue`) |
| **Announcements** | One filterable feed instead of scattered messages |
| **Timetable & Calendar** | KU-style schedule plus deadlines, exams, and events in one view |
| **Resources** | Searchable notes, slides, and past questions per course |
| **Messaging & Study Groups** | Built-in communication and collaboration, no external apps needed |
| **Notifications** | One unified stream instead of per-app alerts |

## HCI Foundation

Designed around **Shneiderman's Eight Golden Rules** (consistency, feedback, error prevention, user control, reduced memory load) plus **User-Centered Design**, **recognition over recall**, and **progressive disclosure** — validated against real fragmentation problems identified in our research.

## Design

- Blue/white minimal SaaS look, Material Design 3 influence
- Inter typeface, Lucide icons, rounded cards, soft shadows
- Localized for **Kathmandu University**: real course codes, semester structure, and timetable format
